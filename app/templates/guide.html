Guide BaseSecrets

BaseSecrets est un portail de gestion des secrets d'entreprise (mots de passe, clés…) stockés dans des enveloppes scellées en coffre-fort. Ce guide couvre l'installation et l'utilisation complète.

Installation

Prérequis

  • Python 3.10 ou supérieur
  • Accès Internet pour le premier lancement (CDN Tailwind CSS)

1. Installer les dépendances

Ouvrez un terminal dans le dossier basesecrets\ et exécutez :

pip install -r requirements.txt

2. Lancer l'application

python run.py

Au premier lancement, la base de données est créée automatiquement et deux comptes par défaut sont initialisés.

3. Accéder à l'application

Ouvrez votre navigateur sur http://127.0.0.1:8000

Module Secrets

Le registre des secrets liste l'ensemble des secrets enregistrés. Chaque secret représente un mot de passe ou une clé stockée dans une enveloppe scellée, elle-même rangée dans un coffre-fort.

Champs d'un secret

IdentifiantClé technique unique et immuable (ex : LDAP_ADMIN)
LibelléNom lisible du secret
Nom techniqueNom du système ou serveur porteur du secret (ex : srv-ldap01)
DomaineDomaine fonctionnel (ex : Active Directory, Réseau)
Coffre fortNom du coffre où est rangée l'enveloppe
N° enveloppeNuméro unique de l'enveloppe actuelle (code-barres scannable)
Scan enveloppePhoto ou scan de la face de l'enveloppe (optionnel, JPG/PNG/PDF)

Recherche

La barre de recherche filtre simultanément sur le libellé, l'identifiant, le nom technique et le numéro d'enveloppe. Les menus déroulants permettent de filtrer par domaine ou coffre. Par défaut, les secrets archivés sont masqués — un lien "Afficher les archivés" permet d'y accéder.

Historique

Chaque modification est tracée dans la fiche du secret : création, modification des informations, changement d'enveloppe, archivage. L'ancienne et la nouvelle valeur sont conservées indéfiniment.

Archivage d'un secret

L'archivage permet de désactiver un secret qui n'est plus utilisé (mot de passe révoqué, système décommissionné…) tout en conservant la traçabilité complète.

Archiver un secret

  1. Ouvrir la fiche du secret
  2. Cliquer sur Archiver (responsable uniquement)
  3. Confirmer l'action — le secret est immédiatement masqué de l'inventaire actif

Désarchiver un secret

Depuis la fiche d'un secret archivé (accessible via le filtre "Afficher les archivés" dans la liste), cliquer sur Désarchiver. Le secret réapparaît dans l'inventaire actif.

Les secrets archivés sont automatiquement exclus de la liste "Secrets non scannés" dans les rapports d'audit.

L'historique complet (créations, changements d'enveloppe, audits passés) reste consultable même après archivage.

Changement d'enveloppe

Lorsqu'un secret est utilisé (ouverture de l'enveloppe), il est reconditionné dans une nouvelle enveloppe avec un nouveau numéro. Ce flux est distinct de la modification classique et génère une entrée spécifique dans l'historique.

Procédure

  1. Ouvrir la fiche du secret concerné
  2. Cliquer sur Changer enveloppe
  3. Scanner le code-barres de la nouvelle enveloppe avec la douchette (ou saisir manuellement)
  4. Optionnel : ajouter une note explicative (raison du changement)
  5. Optionnel : joindre un scan de la nouvelle enveloppe
  6. Cliquer sur Valider le changement
L'ancien numéro d'enveloppe est conservé dans l'historique avec la date, l'heure et le nom de l'opérateur.

Audit annuel

L'audit annuel consiste à ouvrir les coffres, scanner toutes les enveloppes avec la douchette et vérifier que chaque numéro scanné correspond à un secret enregistré en base.

Déroulement

  1. Aller dans Audit annuel → Nouvelle session
  2. Nommer la session (ex : Audit 2025), indiquer l'année
  3. Optionnel : sélectionner le coffre audité dans le menu déroulant — le rapport "manquants" ne montrera alors que les secrets de ce coffre
  4. Valider — l'interface de scan s'ouvre, l'input est automatiquement focalisé
  5. Scanner chaque enveloppe l'une après l'autre avec la douchette
  6. Chaque scan affiche immédiatement le résultat (voir tableau)
  7. À la fin, cliquer sur Clôturer et voir le rapport

Statuts de scan

Conforme Le numéro scanné correspond à un secret en base. Tout va bien.
Inconnu Numéro non trouvé en base. À investiguer : enveloppe non enregistrée ou erreur de saisie.

Rapport

Le rapport de fin de session affiche :

  • Enveloppes conformes — scannées et présentes en base
  • Enveloppes inconnues — scannées mais absentes de la base
  • Secrets non scannés — présents en base mais non scannés (potentiellement manquants), filtrés par coffre si défini et hors archivés

Journal d'activité

Toutes les actions sont tracées automatiquement : connexions, consultations, créations, modifications, changements d'enveloppe et opérations d'audit. Le journal est accessible depuis le menu latéral et peut être filtré par utilisateur ou type d'action.

Gestion des comptes

Accessible depuis le menu Comptes (visible uniquement pour les responsables). Cette section permet de créer, modifier et supprimer les comptes utilisateurs de l'application.

Créer un compte

  1. Cliquer sur Nouveau compte
  2. Saisir un nom d'utilisateur (minimum 3 caractères)
  3. Saisir un mot de passe (minimum 6 caractères)
  4. Sélectionner le rôle : Responsable ou Auditeur

Modifier un compte

Cliquer sur Modifier à côté du compte. Le mot de passe peut être laissé vide pour le conserver sans changement.

Supprimer un compte

Cliquer sur Supprimer. Deux règles s'appliquent :

  • Impossible de supprimer son propre compte
  • Impossible de supprimer le dernier compte responsable
Toutes les opérations sur les comptes sont enregistrées dans le journal d'activité.

Import CSV / Excel

Permet d'alimenter en masse la base depuis un fichier existant (extraction d'Access, Excel RH, etc.).

Procédure

  1. Aller dans Secrets → Importer
  2. Télécharger le modèle CSV si nécessaire
  3. Sélectionner le fichier (.csv, .xlsx ou .xls)
  4. Cliquer sur Importer — un rapport s'affiche immédiatement

Colonnes attendues

ColonneObligatoire
id_secretOui
libelleOui
nom_techniqueOui
domaineOui
coffreOui
num_envelopeOui

Les encodages CSV reconnus : UTF-8, UTF-16, Latin-1, CP1252.

Les lignes en doublon (identifiant ou numéro d'enveloppe déjà présents) sont ignorées sans erreur.

Paramètres

Accessible depuis le menu Paramètres (responsables uniquement). Trois onglets : Général, Thème et Active Directory.

Logo société

Déposez un logo (PNG, JPG, SVG, WEBP — max 2 Mo) depuis l'onglet Général. Il sera affiché sur la page de connexion au-dessus du titre BaseSecrets.

Thème couleurs

Depuis l'onglet Thème, personnalisez l'apparence de l'interface :

  • Couleur principale — fond de la sidebar et de la page de connexion
  • Couleur secondaire — boutons d'action, élément de menu actif, avatar utilisateur

Un bouton "Réinitialiser" remet les couleurs d'origine (bleu nuit / indigo).

SSO Active Directory (LDAP)

Depuis l'onglet Active Directory :

  1. Activer le toggle SSO Active Directory
  2. Renseigner l'adresse du serveur (ex : ldap://dc01.entreprise.local)
  3. Définir le format de connexion : {username}@entreprise.local
  4. Optionnel : renseigner le compte de service (DN + mot de passe) pour les recherches avancées
  5. Optionnel : renseigner l'OU autorisée — seuls les comptes trouvés dans cette OU pourront se connecter (ex : OU=BaseSecrets,DC=entreprise,DC=local)
  6. Optionnel : renseigner le groupe AD requis — l'utilisateur doit en être membre (attribut memberOf)
  7. Choisir le rôle attribué aux nouveaux comptes AD (auditeur par défaut)
  8. Cliquer sur Tester la connexion pour valider l'accès au serveur
  9. Enregistrer

Si l'authentification AD échoue (serveur indisponible, OU non autorisée, groupe manquant), la connexion bascule automatiquement sur le compte local.

Les comptes créés automatiquement depuis AD sont visibles et gérables dans la page Comptes.

Déploiement Docker

Un Dockerfile et un docker-compose.yml sont fournis pour un déploiement conteneurisé.

Démarrage

docker compose up --build -d

L'application est accessible sur http://localhost:8000 depuis n'importe quelle machine du réseau.

Persistance des données

Les données sont stockées dans deux volumes Docker nommés :

  • basesecrets_data — base SQLite et paramètres
  • basesecrets_uploads — scans des enveloppes et logo

Les données survivent aux redémarrages et mises à jour du conteneur.

Variables d'environnement

BASESECRETS_HOSTAdresse d'écoute (défaut Docker : 0.0.0.0)
BASESECRETS_PORTPort (défaut : 8000)
BASESECRETS_RELOADRechargement auto (true en dev, false en prod)

Utilisation de la douchette USB

La douchette code-barres USB fonctionne en mode clavier : elle tape le numéro dans le champ actif puis envoie automatiquement la touche Entrée.

Aucun pilote spécifique n'est requis. Il suffit de :

  1. Brancher la douchette sur un port USB
  2. Ouvrir la page d'audit ou la page de changement d'enveloppe
  3. Le champ de saisie est automatiquement focalisé
  4. Scanner l'enveloppe — la validation est instantanée
Si vous scannez une enveloppe déjà enregistrée dans la session en cours, un message d'avertissement s'affiche et le doublon est ignoré.
